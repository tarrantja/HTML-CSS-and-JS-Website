Security

Purpose

The purpose of this page is to describe how this website was hosted and secured, and why!

Simple Storage Service (S3)

The Amazon Web Services (AWS) S3 service was used as a 'bucket' to hold all the website's html pages, javascript, css and images.

This bucket was configed as a static web application with a domain (click here).

Issue: If you followed the previous link you would have noticed that the connection was not secure. All information is transported as plain text, and is therefore vulnerable to attackers.

This is a limitation of hosting a static web application using S3.

A more robust approach is to enable the website for HTTPS, and is why the additional services below have been used.

Custom Domain Name

Before I go any further I create a custom domain name using NameCheap.

I will also be using NameCheap as by DNS database.

Amazon Certificate Manager (ACM)

For a client to trust my server an SSL Certificate needs to be supplied, that proves I am in control of my domain name.

ACM is used to generate this certificate. I provide them my domain name, an they provide me records to include in my DNS database.

After NameCheap propogates my DNS records, the certificate validates.

CloudFront

AWS's CloudFront is what serves the web content to the public.

All HTTP traffic is redirected to HTTPS, ensuring we always use a secure connection.

This is also where we link our validated SSL Certificate.

CloudFront provides us a domain (link here). Notice that this now provides a secure connection!

Issue: The domain provided above is not very human-friendly.

We solve this by adding another entry to our DNS database, that routes traffic from our custom domain to CloudFront.

Voila!
